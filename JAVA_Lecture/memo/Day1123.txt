클래스 : 설계도
객체 : 물리적인 실제 자료, 인스턴스화한 클래스

추상클래스 : 미완성된 설계도. (대략적으로 그려놓은 설계도)
	-> 미완성된 것이므로 설계를 더 해야함
	-> 부모클래스를 추상클래스로 사용한다. 
	-> 자식클래스는 추상클래스로 인해 사용하도록
	-> 추상클래스의 추상메소드는 반드시 override를 해야한다.
	-> 강제성이 있다. 
	※기존 상속 받은 부모클래스는 강제성이 없으나,
	추상클래스의 메소드들은 강제적으로 override하여 사용해야한다.
	사용이유 : 메소드명을 같은 것으로 지정하여 
	후에 사용하기 편하게 강제적으로 같은이름의 메소드를 만듦

※추상클래스는 일반 클래스처럼 사용해도 된다. 즉, 일반 클래스의 요소가 들어가도 무방
	즉, 추상클래스 = 일반 클래스 + 추상 메소드

final을 가진 클래스는 상속을 하지 않는다.
하지만, abstract class는 final을 사용하지 못한다.
왜? 미완성된 메소드들은 자식을 통해 구현되므로,
final을 사용하지 못한다.
