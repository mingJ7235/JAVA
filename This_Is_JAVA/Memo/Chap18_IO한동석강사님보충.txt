파일 입출력

자바에서 소스코드를 작성해서
자바에서 만든 코드를 외부 파일로 만들고
외부파일을 자바에서 다시 불러와서 출력하는 것임
--------------------------

	Writer (출력)
		-BufferedWriter : 버퍼를 사용해서 파일 작성
		-FileWriter : 경로에 있는 파일 가져오기
			해당 경로에 파일이 없다면 만들어준다. 
		-OutputStreamWriter : byte단위로 파일출력

	Reader (입력)
		-BufferedReader : 버퍼를 사용해서 파일 입력
		-FileReader : 경로에 있는 파일 가져오기
			해당 경로에 파일이 없다면 오류가 난다.
		-InputStreamReader : byte 단위로 파일 입력

-----------------------------------------------

1) Buffer (임시저장소)

-> 일단 이리와!! 개별처리하지 않고 저장하는 임시 저장소 !
-> 저장공간인데, 여기에 개별처리하던걸 일괄 처리하기 위해 존재하는 것임
-> Buffer는 stack구조임
-> Buffering (버퍼링)에서 나오는게 이것임 
	버퍼에 담는중! 아직 flush가 되려면 멀었어! 라는 의미가 버퍼링임
-> flush를 써줘야 버퍼에 있는것들이 출력되는 것임
직접 fulsh를 써줘야함
근데 close를쓰면 flush가 자동적으로 실행댐

	개별 처리가 아닌 일괄 처리로 작업하기 위함
	쌓여 있는 데이터를 목적지로 송출하기 위해서는 flush()라는 메소드로
	버퍼를 비워줘야한다. 
	파일 입출력에서는 close()라는 메소드를 통해서 flush()를
	사용한다. 	
	반드시 사용 후 close() 사용해야 해당 파일에 적용된다.
